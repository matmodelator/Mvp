ЛОГИКА ГЛАВНОГО ОБЩЕГО ЧАТА

✅ mask уникальна сама по себе
✅ color уникален сам по себе
✅ одна и та же mask не может повторяться
✅ один и тот же color не может повторяться

✅ Значит максимум в чате одновременно может быть только 100 users

✅ Потому что
✅ masks = 100
✅ colours = 100
✅ каждому нужен свой уникальный mask
✅ и свой уникальный colour
✅ после 100 пользователей ресурсы заканчиваются

✅ Тогда все остальные идут в waiting room

✅ Правило входа
✅ user выбирает mask
✅ user выбирает colour
✅ система проверяет
✅ свободна ли mask
✅ свободен ли colour
✅ если обе свободны — user попадает в chat
✅ если хотя бы одно занято — в active chat не пускать

✅ Логика waiting room
✅ если в чате уже 100 users — новый user сразу идет в waiting room
✅ если mask занята — waiting room
✅ если colour занят — waiting room
✅ если и mask и colour свободны, но лимит 100 уже достигнут — waiting room

✅ Когда место освобождается
✅ кто-то вышел из чата
✅ его mask освобождается
✅ его colour освобождается
✅ тогда первый подходящий user из waiting room может войти

✅ Но тут важно выбрать принцип
✅ либо waiting room хранит уже выбранные mask и colour
✅ либо user выбирает их только в момент входа в chat

✅ Лучший вариант
✅ в waiting room не закреплять mask и colour
✅ иначе люди начнут блокировать редкие варианты
✅ выбирать mask и colour лучше только когда для user реально есть место в active chat

✅ Что нужно в базе
✅ таблица active_users
✅ user_id
✅ mask_id UNIQUE
✅ color_id UNIQUE

✅ Дополнительно
✅ таблица waiting_room
✅ user_id
✅ joined_at
✅ статус ожидания

✅ Проверка должна быть именно в базе
✅ не только на фронте
✅ иначе два человека могут занять одно и то же одновременно

✅ Итог
✅ максимум 100 users в active chat
✅ каждая mask уникальна
✅ каждый colour уникален
✅ все остальные users — waiting room
✅ при выходе одного user освобождается одно место, одна mask и один colour

✅ Это жесткая и понятная схема
✅ но она очень ограничивает масштаб
✅ больше 100 одновременно у тебя уже никогда не будет

✅ Если цель — сделать именно элитный ограниченный чат, то схема нормальная
✅ если цель — рост, потом она станет узким местом

✅ Для твоего принципа формула такая
✅ unique mask
✅ unique color
✅ max 100 active users
✅ others → waiting room

✅ Самый опасный момент
✅ user может получить mask свободную
✅ но не найти свободный colour
✅ или наоборот
✅ это надо обрабатывать аккуратно

✅ Самое правильное правило
✅ вход в active chat только если одновременно есть
✅ одна свободная mask
✅ и один свободный colour

✅ Иначе
✅ waiting room

✅ Архитектурно идея рабочая
